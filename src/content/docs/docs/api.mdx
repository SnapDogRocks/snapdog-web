## 1. REST API

The SnapDog REST API is hosted on port `5555` by default (configurable via `[http].port`). All requests and responses must utilize the `application/json` Content-Type.

### Authentication & Headers

If token security is activated in `snapdog.toml` (via `[http].api_keys`), clients must include a bearer token in the headers:

```http
Authorization: Bearer <your-configured-api-key>
Accept: application/json
```

#### Authentication Errors
Requests with missing, malformed, or invalid tokens return a `401 Unauthorized` status code with the following JSON schema:
```json
{
  "error": "unauthorized",
  "message": "Invalid or missing API key"
}
```

### Live Interactive API Documentation

If SnapDog is compiled with the `api-docs` feature flag and `api_docs = true` is set in the `[http]` section of your configuration, you can view the interactive developer interface directly in your browser:

* **Scalar Interactive Console:** `http://<device-ip>:5555/docs`
* **Raw OpenAPI Specification:** `http://<device-ip>:5555/api-docs/openapi.json`

The Scalar console allows you to view all endpoints and schemas, and directly perform "Try it out" requests from your browser to test endpoints against live hardware.

## 2. API Endpoints

### Zone Configuration & Playback

#### Get All Zones
* **Endpoint:** `GET /api/v1/zones`
* **Response `200 OK`:**
  ```json
  [
    {
      "index": 1,
      "name": "Living Room",
      "icon": "sofa",
      "playback": "playing",
      "volume": 75,
      "muted": false,
      "source": "subsonic",
      "track": {
        "title": "Acoustic Song",
        "artist": "The Band",
        "album": "Unplugged",
        "duration_ms": 240000,
        "position_ms": 12400,
        "cover_url": "http://192.168.1.10:5555/api/v1/zones/1/cover"
      }
    }
  ]
  ```

#### Get Single Zone Status
* **Endpoint:** `GET /api/v1/zones/{zone_index}`
* **Response `200 OK`:** *(Returns the single zone object depicted above)*
* **Response `404 Not Found`:**
  ```json
  {
    "error": "not_found",
    "message": "Zone index 99 does not exist"
  }
  ```

#### Set Zone Volume
* **Endpoint:** `PUT /api/v1/zones/{zone_index}/volume`
* **Request Body:** Supports absolute integer values (`0` to `100`) or relative string increments (`"+5"`, `"-10"`).
  ```json
  {
    "volume": 80
  }
  ```
  or
  ```json
  {
    "volume": "+5"
  }
  ```
* **Response `200 OK`:**
  ```json
  {
    "index": 1,
    "volume": 80,
    "muted": false
  }
  ```

#### Set Zone Mute State
* **Endpoint:** `PUT /api/v1/zones/{zone_index}/mute`
* **Request Body:**
  ```json
  {
    "muted": true
  }
  ```
* **Response `200 OK`:**
  ```json
  {
    "index": 1,
    "muted": true
  }
  ```

#### Toggle Mute
* **Endpoint:** `POST /api/v1/zones/{zone_index}/mute/toggle`
* **Response `200 OK`:**
  ```json
  {
    "index": 1,
    "muted": false
  }
  ```

#### Play / Pause / Stop Commands
* **Endpoints:** 
  * `POST /api/v1/zones/{zone_index}/play`
  * `POST /api/v1/zones/{zone_index}/pause`
  * `POST /api/v1/zones/{zone_index}/stop`
* **Response `200 OK`:**
  ```json
  {
    "index": 1,
    "playback": "paused"
  }
  ```

#### Playback Mode Control
* **Endpoints:**
  * `POST /api/v1/zones/{zone_index}/next` — Skip to the next track.
  * `POST /api/v1/zones/{zone_index}/previous` — Skip to the previous track.
* **Response `200 OK`:**
  ```json
  {
    "status": "success"
  }
  ```

#### Retrieve Current Cover Art Binary
* **Endpoint:** `GET /api/v1/zones/{zone_index}/cover`
* **Response `200 OK`:** Serves the raw cover art image binary directly.
  * `Content-Type`: `image/jpeg` or `image/png` or `image/webp`.
  * `Cache-Control`: `public, max-age=86400` (1 day).

### DSP & Equalizer

#### Retrieve Zone Equalizer Settings
* **Endpoint:** `GET /api/v1/zones/{zone_index}/eq`
* **Response `200 OK`:**
  ```json
  {
    "enabled": true,
    "preset": "custom",
    "bands": [
      { "frequency": 31, "q": 1.0, "gain": 0.0 },
      { "frequency": 62, "q": 1.0, "gain": 1.5 },
      { "frequency": 125, "q": 1.0, "gain": 3.0 },
      { "frequency": 250, "q": 1.0, "gain": 0.0 },
      { "frequency": 500, "q": 1.0, "gain": -1.0 },
      { "frequency": 1000, "q": 1.0, "gain": 0.0 },
      { "frequency": 2000, "q": 1.0, "gain": 0.0 },
      { "frequency": 4000, "q": 1.0, "gain": 1.0 },
      { "frequency": 8000, "q": 1.0, "gain": 2.0 },
      { "frequency": 16000, "q": 1.0, "gain": 0.0 }
    ]
  }
  ```

#### Modify Equalizer Bands
* **Endpoint:** `PUT /api/v1/zones/{zone_index}/eq`
* **Request Body:**
  ```json
  {
    "enabled": true,
    "bands": [
      { "frequency": 125, "q": 1.0, "gain": 4.5 }
    ]
  }
  ```
* **Response `200 OK`:** Returns the full modified EQ configuration.

#### Load Preset
* **Endpoint:** `POST /api/v1/zones/{zone_index}/eq/preset`
* **Request Body:**
  ```json
  {
    "preset": "bass_boost"
  }
  ```
  *(Supported presets: `flat`, `bass_boost`, `loudness`, `vocals`, `treble_boost`)*
* **Response `200 OK`:** Returns the loaded preset EQ settings.

### Media Streams & Sources

#### Play Arbitrary Stream URL
* **Endpoint:** `POST /api/v1/zones/{zone_index}/play/url`
* **Request Body:**
  ```json
  {
    "url": "https://stream.example.com/live.mp3",
    "title": "My Favorite Web Radio"
  }
  ```
* **Response `202 Accepted`:**
  ```json
  {
    "status": "buffering",
    "url": "https://stream.example.com/live.mp3"
  }
  ```

#### Play Subsonic Track
* **Endpoint:** `POST /api/v1/zones/{zone_index}/play/subsonic/{track_id}`
* **Response `202 Accepted`:**
  ```json
  {
    "status": "buffering",
    "track_id": "9a8b7c6d-5e4f"
  }
  ```

#### Get All Connected Speaker Clients
* **Endpoint:** `GET /api/v1/clients`
* **Response `200 OK`:**
  ```json
  [
    {
      "mac": "02:42:ac:11:00:10",
      "name": "Kitchen Speaker",
      "icon": "cook",
      "zone": "Kitchen",
      "volume": 85,
      "muted": false,
      "latency": 50,
      "max_volume": 100,
      "connected": true
    }
  ]
  ```

#### Set Client Zone Mapping
* **Endpoint:** `PUT /api/v1/clients/{mac}/zone`
* **Request Body:**
  ```json
  {
    "zone": "Living Room"
  }
  ```

  ```json
  {
    "mac": "02:42:ac:11:00:10",
    "zone": "Living Room"
  }
  ```

## 3. WebSocket Interface

The WebSocket interface provides real-time state synchronization. Client applications should open a connection to `ws://<host>:5555/ws`.

### Authentication
If Bearer token authentication is enabled, the token must be passed as a query parameter:
```
ws://<host>:5555/ws?token=<your-api-key>
```

### Event Notification Frames

#### `zone_changed`
Emitted when playback state, track, or position changes.
```json
{
  "type": "zone_changed",
  "zone": 1,
  "playback": "playing",
  "source": "Subsonic",
  "title": "Acoustic Song",
  "artist": "The Band",
  "album": "Unplugged",
  "duration_ms": 240000,
  "position_ms": 12400,
  "volume": 75,
  "muted": false
}
```

#### `zone_volume_changed`
Emitted on volume adjustments.
```json
{
  "type": "zone_volume_changed",
  "zone": 1,
  "volume": 80,
  "muted": false
}
```

#### `client_connected` / `client_disconnected`
Emitted when speaker hardware connects or disconnects from the streaming port.
```json
{
  "type": "client_connected",
  "mac": "02:42:ac:11:00:10"
}
```

## 4. D-Bus Interface (MPRIS2)

For Linux host deployments, SnapDog registers a standard **MPRIS2** media interface on the D-Bus session bus for each zone. 

* **Bus Name:** `org.mpris.MediaPlayer2.snapdog.zone{N}` (where `{N}` is the 1-based index of the zone)
* **Object Path:** `/org/mpris/MediaPlayer2`
* **Implemented Interfaces:**
  * `org.mpris.MediaPlayer2`
  * `org.mpris.MediaPlayer2.Player`

### Control Examples

You can use standard command-line tools like `playerctl` or `busctl` to query and control playback:

```bash
# Toggle play/pause on Zone 1
playerctl -p snapdog.zone1 play-pause

# Skip track on Zone 2
playerctl -p snapdog.zone2 next

# Retrieve current playback volume
playerctl -p snapdog.zone1 volume

# Read active metadata
busctl --user get-property \
  org.mpris.MediaPlayer2.snapdog.zone1 \
  /org/mpris/MediaPlayer2 \
  org.mpris.MediaPlayer2.Player \
  Metadata
```

## 5. Custom Binary Protocol

SnapDog clients (`snapdog-client` / SnapDog OS) communicate with the server over an extended Snapcast binary protocol. Standard Snapcast clients will ignore these frames, while SnapDog hardware processes them directly.

The custom frames utilize Type IDs `10` through `15`:

| ID | Name | Direction | Payload | Description / Format |
|:---:|---|---|---|---|
| **`10`** | `EQ` | Server → Client | JSON | **Equalizer coefficients:** Sends the frequency, Q, and gain parameters for client-side DSP hardware filters. |
| **`11`** | `CORRECTION` | Server → Client | JSON | **Speaker Correction:** Sends the biquad coefficient array generated from the matching Spinorama speaker profile. |
| **`12`** | `PREEMPT` | Server → Client | None | **Preempt Fade-out:** Signals the client to fade out immediately to allow another source to take over without clicks. |
| **`13`** | `CONTROL` | Client → Server | Binary | **Client Control Commands:** Transmits physical button events (e.g. Play, Pause, Next) pressed on client hardware back to the server. |
| **`14`** | `METADATA` | Server → Client | JSON | **Track Info:** Sends the title, artist, album, and playing state for display on client OLED/LCD hardware. |
| **`15`** | `COVER_ART` | Server → Client | Binary | **Direct Image Stream:** Broadcasts the raw cover art image binary (JPEG/PNG) in chunks directly to the client's screen. |
